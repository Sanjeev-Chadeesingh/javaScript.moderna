SCOPE

Think of scope as variable visibility. Everything is scoped in javascript, meaning everything has a level of access, and these access points are limited by their visibility.

Function Scope → defined within the function, meaning, visible only within the function.

function lol() {

   let person = 'Tom';

};

In the above function, person is scoped to the lol function, and can't be accessed outside of the function

Block Scope → defined within the block.

let radius = 8;

if(radius > 0) {

    const PI = 3.14;

    let circ = 2 * PI * radius;

};

PI+circ are scoped within the block of the if statement above, they aren't accessible anywhere else

if(true) {

   var animal = 'eel';

};

console.log(eel);

The above line works, can log animal outside of the block. After block, it logs animal fine. Before the block, it logs undefined. let+const DO NOT WORK. This is one example of where they differ from var.

let is good for LOOPS, it scopes the iterator to the block. var based blocks allow access outside of said blocks. The iterator is always available, regardless of where you are.

let+const cannot be reused if in the same block, whereas var can be.

function doubleArr(arr) {

   const result = [];

   for(let num of arr) {

       let double = num * 2;

       result.push(double);

   }

return result;

};

console.log(doubleArr([1, 2, 3]));

If had set vars instead of let+const, could have logged things inside and out of blocks within both functions and loops

You have access in a block to variables created outside the block.

Could use var instead of let when naming double, and you'd have access to double outside the block it is in.

const can't be reassigned, nor can it be redeclared. Can change or mutate let, but both are block scoped.

var you can basically do whatever with. VAR IS NOT BLOCK SCOPED

Lexical Scope:

When we have nested functions, they are lexically bound, meaning they are bound to the scope of their parents or outer functions, but parents' don't have access to the scope of their children or outer functions.

Access moves one block level up at a time. If they don't find what they need one block out, they move to the next block out.

function outer() {
   let movie = 'Amadeus';
   function inner() {
      let movie = 'The Shining';
      function extraInner() {
         console.log(movie.toUpperCase());
         }
      extraInner();
      };
    inner();
     };
outer();

Above line logs 'The Shining'
